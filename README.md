# Исследование хранилищ для UGC

Сравнение PostgreSQL и MongoDB для хранения пользовательского контента
(оценки, рецензии, лайки и закладки).

В рамках исследования сравниваются **PostgreSQL** и **MongoDB**.

**PostgreSQL** выбран как базовое реляционное хранилище, которое уже
используется в
проекте и поддерживает строгую консистентность, индексы, транзакции и
SQL-запросы.

**MongoDB** выбрана как документоориентированное хранилище, подходящее для
UGC-данных: лайков, рецензий, голосов за рецензии и закладок.

# Цель исследования

Сравнить PostgreSQL и MongoDB в сценариях хранения и чтения UGC-данных:

- оценок фильмов;
- рецензий;
- оценок рецензий;
- закладок.

## Модель данных

### Оценки фильмов:

```json
{
  "user_id": "uuid",
  "movie_id": "uuid",
  "score": 10,
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

Индексы:  
`user_id`  
`movie_id + score`

Уникальный индекс:  
`user_id + movie_id`

### Рецензии

```json
{
  "review_id": "uuid",
  "user_id": "uuid",
  "movie_id": "uuid",
  "author": "string",
  "review_text": "text",
  "movie_score": 10,
  "review_likes": 0,
  "review_dislikes": 0,
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

Индексы:  
`user_id`  
`movie_id + review_likes`  
`movie_id + created_at`

### Оценки рецензий

```json
{
  "user_id": "uuid",
  "review_id": "uuid",
  "score": 10,
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

Индексы:   
`user_id`  
`review_id + score`

Уникальный индекс:  
`user_id + review_id`

#### Закладки

```json
{
  "user_id": "uuid",
  "movie_id": "uuid",
  "created_at": "datetime"
}
```

Индекс:  
`user_id + created_at`

Уникальный индекс:  
`user_id + movie_id`

## Сценарии чтения

1. Получение списка оценок пользователя.
2. Подсчёт количества лайков и дизлайков фильма.
3. Расчёт средней пользовательской оценки фильма.
4. Получение списка закладок пользователя.
5. Получение списка рецензий фильма с сортировкой по количеству лайков.

## Объём тестовых данных

Для моделирования нагрузки принято:

- 1 000 000 пользователей;
- 10 000 фильмов;
- каждый пользователь оценивает в среднем 10 фильмов;
- каждый пользователь добавляет в закладки в среднем 10 фильмов;
- каждый пользователь пишет рецензию;
- каждый пользователь оценивает в среднем 10 рецензий.

На основе этих допущений получены следующие объёмы данных:

- 10 000 000 оценок фильмов
- 10 000 000 закладок
- 10 000 000 оценок рецензий
- 1 000 000 рецензий

## Вывод

Обе базы данных успешно справляются с нагрузкой на объёме более 30 миллионов
записей и обеспечивают время чтения значительно ниже целевого значения 200 мс.

MongoDB показала:

- более быструю загрузку данных;
- немного более быстрые операции чтения в большинстве сценариев;
- естественное представление UGC в виде документов;
- удобные механизмы горизонтального масштабирования через шардинг.

PostgreSQL показал:

- значительно более быстрое создание индексов;
- предсказуемые планы выполнения запросов;
- высокую производительность даже на объёме десятков миллионов записей.

Для хранения UGC-данных (оценки, рецензии, лайки, закладки) выбрана MongoDB,
поскольку сценарии доступа не требуют сложных JOIN-запросов, а возможность
горизонтального масштабирования является важным преимуществом при росте объёма
пользовательского контента.

## Как запустить проект:

1. Клонировать репозиторий и перейти в него в командной строке:

```commandline
git clone https://github.com/vivat-7on/ugc_benchmark.git
```

```commandline
cd ugc_benchmark
```

2. Создать и активировать виртуальное окружение:

(macos)

```commandline
python3.11 -m venv .venv
```

```commandline
source .venv/bin/activate
```

3. Установить зависимости из файла requirements.txt:

```commandline
pip install --upgrade pip
```

```commandline
pip install -r requirements.txt
```

4. Развернуть инфраструктуру:

```commandline
docker compose up -d --build
```

5. Объединить компоненты MongoDB в кластер:

```commandline
./scripts/init_mongo_cluster.sh
```

6. Сгенерировать и загрузить данные:

> Генерация полного набора данных (31 млн записей) может занять
> продолжительное время и требует значительного объёма свободной памяти
> и дискового пространства.

```commandline
python generate_data.py
```

7. Запустить тест чтения:

```commandline
python benchmark_read.py
```
